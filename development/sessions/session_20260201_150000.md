### 1. Fixed Customer Points Calculation in Sales
**Problem**: Points were not being added to `accumulated_points` when customers made purchases. The backend was relying on the POS to send pre-calculated points (which was always 0).

**Solution**: Backend now calculates points server-side:
- Sums subtotals from items where `creditable_to_points = 1`
- Multiplies by `customer.points` (the multiplier rate)
- Adds result to `customer.accumulated_points`

## Technical Notes
- Broadcasting requires Pusher configuration in `.env`:
  ```
  BROADCAST_CONNECTION=pusher
  PUSHER_APP_ID=your-app-id
  PUSHER_APP_KEY=your-app-key
  PUSHER_APP_SECRET=your-app-secret
  PUSHER_APP_CLUSTER=ap1
  ```
- Higher access requests expire after 2 minutes
- Scheduled command `higher-access:expire` runs every minute to clean up expired requests
