## Open security audit findings

Full audit ran 2026-05-11 via the general-purpose agent. Tranche 1 (Critical) is shipped. The rest are open:

### Critical â€” ALL FIXED in commit `3678e47`
- C-01: `POST /superadmin/register` public âœ…
- C-02: `POST /admin/register` public âœ…
- C-03: `POST /api/v1/register` + `POST /api/v1/image-updater/register` public âœ…
- C-04: `GET /fix/database` unauthenticated `REPAIR TABLE` âœ…

### High â€” Open
- H-01: Cross-tenant IDOR on route-model-bound admin/POS controllers (Item, Customer, Voucher, SpecialCustomer, WholesalePriceTier, Sale::refund, ShopAnnouncement, CustomerCredit). Each loads by ID without checking `$model->user_id === auth()->user()->user_id`. **Folded into the multi-tenancy plan's Tranche C.**
- H-02: SVG accepted as item image â€” same stored-XSS class as the already-patched avatar bug. `ItemController::store/update` and `ProductImageController` both have `mimes:jpg,png,jpeg,gif,svg`. Drop `svg`, match the avatar fix.
- H-03: Stored XSS in admin DataTables â€” values like `$customer->name`, `$item->sold_by`, etc. interpolated into action HTML without `e()`, then `rawColumns([...])`. Audit every `rawColumns` and wrap interpolated values with `e()`.
- H-04: Server-trusted financial fields in POS sale (`profit`, `total`, `line.*.cost`, `line.*.price`, `points_used`, `voucher_discount`). Submitting `total=1, profit=1000000` is accepted and grants loyalty points. Server must recompute everything from `Item.price/cost` by id.
- H-05: Voucher TOCTOU â€” concurrent `apply()` calls when `used_count = max_uses - 1` both pass the gating check. Wrap in `DB::transaction` + `lockForUpdate`. Add unique index on `(voucher_id, sale_id)` to voucher_usages.
- H-06: Desktop attendance API entirely unauthenticated. `/api/desktop/v1/attendance/*` group has no middleware. Anyone can clock employees in/out and enumerate names by barcode.
- H-07: Higher-access requests impersonatable. `HigherAccessController::store` accepts `user_id`/`store_id`/`pos_id` as request fields with only `exists:` validation; no check that they belong to the auth user's tenant.
- H-08: No rate limiting on any login route. `admin`, `superadmin`, POS, mobile, customer, image-updater logins â€” all wide open to brute-force.
- H-09: `HomeController::default` (admin dashboard) queries Sales/Expenses with no tenant filter. **Folded into the multi-tenancy plan's Tranche C.**

### Medium â€” Open
- M-01: Mass-assignment exposure on `User.$fillable` (`role_id`, `user_id`, `is_customer`, `status`, `code`, `rate`) and `Customer.$fillable` (`points`, `accumulated_points`, `credit_limit`, `credit_balance`, `is_wholesale`, `user_id`).
- M-02: Customer API tokens bypass email verification.
- M-03: `/admin/tests/sms/send` has no role gate, no rate limit. Cost-amplification vector against the merchant's SMS account.
- M-04: Empty `VerifyCsrfToken` middleware â€” unused in Laravel 12 but adds confusion.
- M-05: Duplicate `/shop` route in `web.php` (bypasses `LogShopVisit` middleware on `ecommerce.php`).
- M-06: Bank deposit/withdrawal/transfer balance update not row-locked. Concurrent operations corrupt `balance_before` / `balance_after` history.
- M-07: Widespread `{!! Form::* !!}` pattern in superadmin/admin settings views â€” currently safe but pattern invites XSS.
- M-08: `APP_DEBUG=true` in committed `.env.example`. Production deploys derived from it would leak stack traces.

### Low â€” Open
- L-01: Email enumeration on `/api/v1/auth/higher_access` (different message for unknown email vs. wrong password).
- L-02: `verifyUiniqid` returns role flag values without allowlist validation on `$type`.
- L-03: Public `/api/v1/contact` has no rate limit.
- L-04: `env()` calls outside config in `SmsController` and `NewCustomerJob`.
- L-05: Date string interpolation in `SuperAdmin\AdjustmentController::readings()` raw SQL.

### Info â€” partially closed by Auditable trait work
- I-01: `created_at` fillable on Sale model.

## Repo conventions worth knowing

- Sail is mandatory for everything. `vendor/bin/sail artisan ...`, never raw `php artisan`.
- Pint enforces `test_snake_case` method naming.
- Tests use PHPUnit, not Pest.
- `vendor/bin/sail artisan test --filter=name_of_test` for focused runs.
- Test DB safety: `phpunit.xml` forces `DB_DATABASE=testing`; `tests/bootstrap.php` asserts before any test runs.
- Mail in tests: use `Mail::assertSent` (not `assertQueued`); commands use `Mail::to()->sendNow()`.
- Branches: only `dev` and `main` live in the remote at any time. Feature branches get created, tested, merged FF, deleted.
- Memory: auto-memory lives at `~/.claude/projects/-Users-richardleosala-Projects-RLCPS-apex-backend/memory/` and survives across sessions; references in `MEMORY.md` map to per-topic files.
